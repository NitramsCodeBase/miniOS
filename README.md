# miniOS

“miniOS” is a private fun and learning project written in a combination of **Assembly** and **C**. The project was created to emulate simple command-line shell interactions similar to early operating systems such as **MS-DOS**.

The primary goal of the project is to better understand how older computer systems from the late 1980s and 1990s booted and interacted with hardware on a low level. The project explores concepts such as bootloaders, memory access, basic shell environments, and low-level system programming.

A custom `boot.asm` bootloader was used to initialize the system during startup. The project was partially developed with the assistance of AI tools, especially for understanding Assembly syntax, bootloader structure, and generating some low-level code examples. The remaining implementation and experimentation in C were used as a hands-on learning process to better understand operating system fundamentals and low-level software architecture.

## Features

- Custom Assembly bootloader (`boot.asm`)
- Simple command-line shell interactions
- MS-DOS inspired terminal behavior
- Low-level system initialization
- Combination of Assembly and C programming
- Experimental operating system structure
- Educational boot process emulation
- Keyboard input handling
- Basic screen output functionality
- Learning-focused low-level programming environment

## Technologies Used

- Assembly (x86)
- C
- Bootloader programming
- Low-level memory interaction
- BIOS interrupt concepts
- Operating system fundamentals

## Background

This project was created as a personal exploration into retro-computing and operating system development. The main purpose was to understand how classic machines booted and operated before modern operating systems became highly abstracted.

The project also served as a practical learning experience in low-level programming, Assembly language, and system architecture. AI-assisted development was used as a supportive learning tool for understanding certain concepts and generating reference implementations while continuing to experiment and learn through hands-on coding.
